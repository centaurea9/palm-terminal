# 多级菜单 — 分步开发 & 调试计划

> 每步只做一件事，编译烧录验证通过后再进下一步。

## 当前基准（main.cpp 不动的内容）

```
屏幕: 284×76 横屏  |  左面板: 0~89px  |  右面板: 90~284px（菜单区）

右面板 3 行固定位置（选中项左移 10px 突出）:
  行1（非选中）: y=7,  x=185 → screen_draw_text(185, 7,  "...")
  行2（选中）  : y=31, x=175 → screen_draw_text(175, 31, "...")  ← 黄框 + ► 焊死在这行
  行3（非选中）: y=56, x=185 → screen_draw_text(185, 56, "...")

黄框: drawRect(94, 24, 186, 22, TFT_YELLOW)   ← 固定在行2区域
► : 位置在 x=175 左侧（具体 x 待定）            ← 仅在选中行显示
```

## Step 1 — 新建 menu.h，只定义数据结构

**目标**: 文件存在，编译通过，不接入 main。

### 文件: `src/menu.h`

```cpp
#ifndef MENU_H
#define MENU_H

#include <stdint.h>

// 前向声明
struct MenuItem;

struct MenuItem {
    const char *title;        // 右侧菜单文字（UTF-8）
    const uint8_t *left_icon; // 左侧图标（1-bit bitmap，预留，nullptr 就不画）
    int icon_w, icon_h;       // 图标尺寸
    const char *left_text;    // 左侧显示文字
    MenuItem *parent;         // 父菜单（顶层填 nullptr）
    MenuItem **children;      // 子菜单数组（nullptr 表示叶子节点）
    int child_count;          // 子菜单数量
};

// 菜单树根
extern MenuItem *menu_root;

#endif
```

### 调试

```bash
pio run   # 仅验证编译通过
```

✅ 通过标志: 编译零错误零警告。

## Step 3 — main 接入：画出第一屏菜单（静态，无滚动）

**目标**: 右侧 3 行显示菜单项，`►` 固定在中间行，黄框画出来，能看。

### main.cpp 改动

- `#include "menu.h"`
- 选一个"当前节点"（比如 `current_menu = menu_root`）
- 渲染 `current_menu->children[0~2]` 到 3 行
- 选中永远是第 2 行 → `children[1]`
- 左侧暂时不动（保持星期天+时间）

### 调试

- 烧录，看屏幕：3 行文字 + 第 2 行有 `►` + 黄框
- 确认文字位置、字号和你预期一致

✅ 通过标志: 屏幕显示 3 项菜单，第 2 行标记选中。

---

## Step 4 — 串口读键，上下移动游标

**目标**: 在 Serial Monitor 敲 W/S，游标在 3 行内移动（列表不变，`►` 换行）。

### main.cpp 改动

- `loop()` 中读 `Serial.available()`，解析单字符 W/S
- 维护 `int cursor = 0`（在 3 行中的位置）
- W: `cursor--`，S: `cursor++`，边界钳制
- 根据 `cursor` 决定哪行画 `►`

### 调试

| 串口输入 | 预期效果 |
|:---|:---|
| `S` | `►` 从行1→行2 |
| `S S` | `►` 到行3 |
| `S S S` | `►` 停在行3不变 |
| `W W W` | `►` 停在行1不变 |

✅ 通过标志: `►` 在 3 行间自由移动，不越界。

---

## Step 5 — 滚动逻辑（游标在行3再按 S，列表滚一行）

**目标**: 数据不止 3 项时（比如 5 项），列表整体滚动。

### main.cpp 改动

- 新增 `int scroll_offset = 0`（当前列表从第几项开始显示）
- 渲染 `children[scroll_offset + 0~2]`
- 游标到行3 + S → `scroll_offset++`（游标留在行3）
- 游标到行1 + W → `scroll_offset--`（游标留在行1）
- 游标在中间行 → 移动游标，不滚列表
- 边界条件：第一条/最后一条到位后停止

### 调试

```
主菜单有 3 项（不够滚）
  验证: 游标移动正常，scroll_offset 始终为 0

临时加一项做成 4 项来测:
  初始: [1] [►2] [3]     scroll=0, cursor=1
  S:    [2] [►3] [4]      scroll=1, cursor=1  
  S:    停住（已到底）
```

✅ 通过标志: 多于 3 项的列表可以滚动，边界不越界。

## Step 6 — Enter 确认，进入子菜单

**目标**: 按 Enter，current_menu 切换到子节点，重新渲染。

### main.cpp 改动

- 解析 Enter 键（`\r` 或 `\n`）
- `selected_index = scroll_offset + cursor`
- `next = current_menu->children[selected_index]`
- 如果 `next->children != nullptr`：`current_menu = next`，`cursor = 0`，`scroll_offset = 0`

### 调试

| 操作 | 预期 |
|:---|:---|
| 选"干员寻访"→Enter | 右侧显示子菜单："公开招募""限定寻访" |
| 只有 2 项 | 第 3 行空白（不崩溃） |
| 叶子节点按 Enter | 无视（暂时） |

✅ 通过标志: 可进入有子菜单的项，界面刷新正确。

---

## Step 7 — Q 返回上级

**目标**: 按 Q 回到父菜单。

### main.cpp 改动

- 解析 Q 键
- 如果 `current_menu->parent != nullptr`：`current_menu = current_menu->parent`，复位游标
- 如果已经是顶层：不响应

### 调试

| 操作 | 预期 |
|:---|:---|
| 入"系统设置" → Q | 回到主菜单 |
| 主菜单按 Q | 没反应 |

✅ 通过标志: Q 键正确返回上级。

---

## Step 8 — 左侧面板联动

**目标**: 切换选中项时，左侧显示对应 `left_text`（以及将来 `left_icon`）。

### main.cpp 改动

- 选中项变化时，获取 `current_menu->children[selected_index]->left_text`
- 用 `screen_draw_text` 画到左侧对应位置
- 顶层默认：无选中时保持星期天+时间

### 调试

- W/S 切换选项，左侧文字跟着变
- 进入子菜单，左侧显示父信息
- 返回顶层，左侧恢复星期天+时间

✅ 通过标志: 左侧跟随选中实时更新。

## 开发节奏建议

| 步 | 预估时间 | 关键风险 |
|:---|:---|:---|
| Step 1~2 | 5 min | 数据结构后续够不够用 |
| Step 3 | 10 min | 坐标对齐 ← **最关键** |
| Step 4~5 | 15 min | 滚动边界逻辑 |
| Step 6~7 | 10 min | 父子指针正确性 |
| Step 8 | 10 min | 左侧刷新不闪屏 |
| Step 9 | 10 min | 边界 case |

> **原则**: 每步做完立刻烧录看效果，不要攒到最后。视觉反馈比串口日志快 10 倍。
